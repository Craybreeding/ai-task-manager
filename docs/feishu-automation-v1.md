# AI Captain 飞书自动化流程 V1

## 1. 目标

只做最有价值的 4 条自动化：

1. GitHub 任务同步
2. 项目周摘要生成
3. DDL / 阻塞提醒
4. 运维异常提醒

V1 不做复杂智能体编排，先把项目管理闭环跑通。

## 2. 数据责任边界

### 自动同步维护

- GitHub 任务状态
- GitHub assignee
- GitHub 更新时间

### Captain 人工维护

- 当前在做
- 当前卡点
- 最新反馈
- 下个检查点
- 风险灯色

### 运营/技术负责人维护

- Eval 得分
- 运维事件
- 值班信息

## 3. 自动化 1：GitHub -> 飞书任务同步

### 输入

- GitHub Project items
- Issue / PR metadata

### 输出

- 写入 `任务表`

### 最低同步字段

- 任务标题
- 当前状态
- 负责人
- GitHub 链接
- 更新时间
- 关联项目

### 更新频率

- 每 30 分钟

## 4. 自动化 2：每周项目摘要

### 输入

- `项目主表`
- `周更新表`
- `Eval 表`
- `运维事件表`

### 输出模板

- 项目名
- 当前在做
- 当前卡点
- 最新反馈
- 本周推进
- 下个检查点
- 是否需要升级

### 频率

- 每周五 18:00

## 5. 自动化 3：DDL / 阻塞提醒

### 触发条件

- 距DDL <= 7 天
- 当前状态 = 红灯
- 存在关键路径阻塞任务

### 提醒对象

- Captain
- Sponsor

### 提醒内容

- 项目名
- DDL
- 当前卡点
- 当前 owner
- 下个检查点

## 6. 自动化 4：运维异常提醒

### 触发条件

- `运维事件表` 新增 P1 / P2

### 提醒对象

- 值班人
- Captain

### 提醒内容

- 项目名
- 事件类型
- 严重等级
- 当前状态
- 恢复负责人

## 7. V1 不做的事

先明确砍掉，避免范围膨胀：

- 不做独立权限系统
- 不做自定义前端 BI
- 不做复杂报表中心
- 不做多平台双向全量同步
- 不做自动判断所有风险灯

## 8. 实施建议

第一步先把 `项目主表 + 周更新表 + 任务表` 建出来。

这是最短闭环，因为你马上就能回答：

- 项目在哪个阶段
- 当前在做什么
- 卡点是什么
- 最近反馈是什么
- 谁负责
- 下个检查点是什么

这比先做额外看板更值。
